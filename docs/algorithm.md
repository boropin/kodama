# 解析・判定仕様 v0.1

## 音声処理

1. Web Audio APIで音声ファイルをデコードする。
2. 複数チャンネルの場合はモノラルへ平均化し、DC成分を除く。
3. 約6 msの短時間RMS包絡を3 ms間隔で計算する。
4. 包絡の20パーセンタイルを雑音床、96パーセンタイルを高レベル基準とする。
5. 両者の間に適応しきい値を置き、160 msの不応期を設けて打音ピークを検出する。
6. 各打音の直後約85 msにHann窓を掛け、radix-2 FFTを行う。
7. 80〜900 Hzの最大エネルギー周波数を主周波数とする。
8. 80〜1,200 Hzのエネルギーに対する80〜400 Hzの比を低音比率とする。
9. 打音後の短時間RMSがピークの20%未満で一定時間続くまでを余韻の推定値とする。

## 品質指標

- 打音が3回未満なら品質を大きく下げる。
- 15回を超える場合は、雑音を打音として検出した可能性を示す。
- 高レベルと雑音床の比が小さい場合は、周囲雑音が大きいとみなす。
- 各打音の主周波数の変動係数が大きい場合は、叩き方のばらつきを示す。

品質は主に「暫定確信度」へ反映し、成熟度スコアを直接大きく動かさない。

## 暫定成熟度スコア

`rule-v0.1` は合計100点で、受粉後日数25点、ひげ15点、トライコーム15点、模様15点、打音30点とする。

打音は主周波数55%、低音比率30%、余韻15%で構成する。低い主周波数、低音比率の増加、余韻の増加を成熟方向として扱うが、これは未校正の仮説である。

## 出力と再現性

JSONには次を保存する。

- 入力値と日付
- 音声ファイルのメタ情報
- 打音位置、打音別特徴、平均特徴
- 検出感度としきい値
- 品質指標
- 各カードの寄与点
- 判定、暫定確信度、理由
- スキーマとモデルのバージョン

音声本体はJSONに含めない。音声とJSONを同じ個体IDで管理する仕組みは次版で追加する。
